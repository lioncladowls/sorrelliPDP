This guide documents the full implementation spec for the Sorrelli cart surfaces — the dedicated /cart page and the slide-out drawer. The audience is a Shopify developer porting the mockup into Liquid. Each section covers HTML structure, CSS classes, JavaScript patterns, and Shopify-specific notes.

1. Architecture Overview

1

Two Surfaces, One Cart State

The Two Surfaces

The cart system has two distinct surfaces, each with a different job:

  • Cart page (/cart) — the deliberation surface. 60/40 layout with a full order summary sidebar, promo codes, express checkout, delivery estimate, cross-sell, and gift options. Triggered by clicking the cart icon in the header.
  • Slide-out drawer — the momentum surface. Stripped to essentials: items, shipping progress, subtotal, CTA. Triggered automatically when an item is added to cart.

Trigger Pattern

  • Cart icon in header links to the /cart page (standard anchor navigation).
  • Add to Cart button on PDP opens the drawer via JavaScript and does not navigate away from the page.
  • The drawer can also be opened programmatically via URL param: ?drawer=1.

Shared Cart State

Both surfaces read from and write to the same cart state. Any qty change or item removal on the cart page is reflected in the drawer and vice versa. The shared key is the data-item-id attribute on each line item element. In Shopify, cart state is managed via the Cart AJAX API; in the mockup, it is managed in-memory via JavaScript.

File Structure

In the HTML mockup, both surfaces live in a single self-contained file (cart.html) for preview purposes. In Shopify, they map to separate Liquid templates:

  • templates/cart.liquid (or templates/cart.json + section) — the /cart page
  • sections/cart-drawer.liquid — the slide-out drawer, included in theme.liquid so it's available site-wide
  • assets/cart.js — shared cart JavaScript (recalculate, sync, drawer open/close)

Single source of truth: all cart state mutations (qty changes, removals) call recalculate() and sync both surfaces before the DOM settles.

Dynamic Behavior

  • Threshold constant: FREE_SHIP_THRESHOLD = 150
  • Fill width: Math.min(100, (total / 150) * 100) + '%'
  • Below threshold: truck SVG, copper text, rose-gold fill, message shows dollar amount remaining
  • At/above threshold: checkmark SVG, blue text, blue fill, "Free shipping unlocked!"
  • CSS transition on the fill element: width 0.4s ease — the smooth fill animation is itself a conversion nudge

CSS for Fill Transition

.free-shipping-bar__fill,
.cart-drawer__shipping-fill {
  height: 100%;
  background: var(--color-rose-gold);
  border-radius: inherit;
  transition: width 0.4s ease;
}

/* Qualified state: swap fill color to blue */
.free-shipping-bar--qualified .free-shipping-bar__fill {
  background: var(--color-accent-blue);
}

Both bars must update together on every recalculate() call. They read the same total but update independently via their own element IDs.

Cart Item HTML Structure

Each cart line item uses data attributes as the source of truth for JS calculations. Set these from Liquid when rendering:

  • data-cart-item — selector used by recalculate() to iterate all items

6. Order Summary Sidebar

6

The Sidebar Conversion Flow

Order of Elements (Top to Bottom)

The sidebar order is a deliberate conversion sequence. Do not rearrange elements without reviewing the design philosophy rationale.

  1. Express Checkout Buttons — Shop Pay (purple), PayPal (gold), Google Pay (black). Use Shopify's {% render 'dynamic-checkout-buttons' %} or the additional_checkout_buttons object.
  2. "or" divider — simple horizontal rule with centered text label.
  3. Order Summary card — subtotal, clearance savings (conditional), shipping, tax, total.
  4. BNPL line — "or 4 interest-free payments of $XX.XX with [Afterpay logo]". Amount = total / 4.
  5. Checkout CTA button — "Continue to Checkout", full-width, primary. This is the IntersectionObserver target for the sticky mobile bar.
  6. Delivery estimate — "Estimated delivery: [date range]". Calculated dynamically in JS (see Section 9).
  7. Trust strip — three items: "Free Returns", "Secure Checkout", "40+ Years of Quality". SVG icons + text.
  8. Promo code — collapsed behind "Have a promo code?" toggle link. Expands to text input + Apply button on click.

7. Drawer Footer

7

Stripped to Essentials

Drawer Structure (Top to Bottom)

The drawer has three zones: header (close button + shipping bar), scrollable item list, and sticky footer.

  1. Drawer header — "Your Bag" title, X close button, compact shipping bar
  2. Scrollable item list — same .cart-item structure as cart page, with data-drawer-item and data-item-id
  3. Sticky footer — the following in order:
    1. Subtotal row (label + amount)
    2. Shipping row ("Shipping" + "$9.95" or "FREE")
    3. Tax line ("Tax calculated at checkout" — static, muted)
    4. BNPL line ("or 4 payments of $XX.XX with [Afterpay logo]")
    5. Checkout CTA ("Go to Checkout")
    6. Trust line (lock SVG + "Secure Checkout · Free Returns" — single line, compact)
    7. Gift text link ("Sending as a gift? See options →" → links to /cart)
    8. View Full Cart link → /cart

What Is Intentionally NOT in the Drawer

Omitted from drawer by design

  • Cross-sell / "You may also like" — without AI-curated matching (Rebuy), generic cross-sell violates the drawer's confirmation-surface principle; lives on /cart page only
  • Gift checkbox — replaced with a single text link ("Sending as a gift? See options →") to reduce cognitive load for the 80%+ self-purchasers; full gift card lives on /cart page
  • Promo code input — opens keyboard on mobile, sends 8% of users off-site to find codes
  • Express checkout buttons — checkout page will present them; drawer is not the place
  • Delivery estimate — too much detail for a momentum surface
  • Full trust badge strip — condensed to a single trust line below CTA

9. Dynamic State & JavaScript

9

Core JS Patterns

Constants

  • FREE_SHIP_THRESHOLD = 150
  • Shipping cost when not free: $9.95 (hardcoded display; actual charge from Shopify rates)

The recalculate() Function

The heart of cart state management. Called on every qty change and item removal. Updates all dynamic values across both surfaces in a single pass.

var FREE_SHIP_THRESHOLD = 150;

function recalculate() {
  var total = 0;
  var count = 0;
  var clearanceSavings = 0;

  // 1. Iterate all cart items and accumulate totals
  document.querySelectorAll('[data-cart-item]').forEach(function(item) {
    var price    = parseFloat(item.dataset.itemPrice) || 0;
    var origPrice = parseFloat(item.dataset.itemOriginalPrice) || 0;
    var qty      = parseInt(item.querySelector('[data-qty-val]').textContent) || 1;

    total += price * qty;
    count += qty;
    if (origPrice > 0) clearanceSavings += (origPrice - price) * qty;
  });

  // 2. Shipping
  var shipFree = total >= FREE_SHIP_THRESHOLD;
  var shipCost = shipFree ? 0 : 9.95;
  var grandTotal = total + shipCost;

  // 3. Update shipping bars
  var pct = Math.min(100, (total / FREE_SHIP_THRESHOLD) * 100);
  var remaining = (FREE_SHIP_THRESHOLD - total).toFixed(2);

  document.getElementById('page-ship-fill').style.width = pct + '%';
  document.getElementById('drawer-ship-fill').style.width = pct + '%';

  if (shipFree) {
    document.getElementById('page-ship-msg').textContent = 'Free shipping unlocked!';
    document.getElementById('drawer-ship-msg').textContent = 'Free shipping!';
    // Swap truck → checkmark SVG, add --qualified class
  } else {
    document.getElementById('page-ship-msg').textContent =
      '$' + remaining + ' away from free shipping';
    document.getElementById('drawer-ship-msg').textContent =
      '$' + remaining + ' away';
    // Swap checkmark → truck SVG, remove --qualified class
  }

  // 4. Update order summary
  document.getElementById('summary-subtotal').textContent =
    '$' + total.toFixed(2);
  document.getElementById('summary-shipping').textContent =
    shipFree ? 'FREE' : '$9.95';
  document.getElementById('summary-total').textContent =
    '$' + grandTotal.toFixed(2);

  // 5. Clearance savings row (show/hide)
  var savingsRow = document.getElementById('savings-row');
  if (clearanceSavings > 0) {
    savingsRow.hidden = false;
    document.getElementById('summary-savings').textContent =
      '−$' + clearanceSavings.toFixed(2);
  } else {
    savingsRow.hidden = true;
  }

  // 6. BNPL amounts
  var bnpl = (grandTotal / 4).toFixed(2);
  document.getElementById('bnpl-amount').textContent = '$' + bnpl;
  document.getElementById('drawer-bnpl-amount').textContent = '$' + bnpl;

  // 7. Sticky mobile bar
  document.getElementById('sticky-total').textContent = '$' + grandTotal.toFixed(2);

  // 8. Drawer totals
  document.getElementById('drawer-subtotal').textContent = '$' + total.toFixed(2);
  document.getElementById('drawer-shipping').textContent = shipFree ? 'FREE' : '$9.95';

  // 9. Heading counts
  document.getElementById('cart-count').textContent = count;
  document.getElementById('drawer-count').textContent = count;
}

Cart ↔ Drawer Sync

Both surfaces use data-item-id as the shared key. When a qty changes on one surface, the other is updated immediately.

// Sync a qty change FROM the cart page TO the drawer
function syncToDrawer(itemId, qty) {
  var drawerItem = document.querySelector(
    '[data-drawer-item][data-item-id="' + itemId + '"]'
  );
  if (!drawerItem) return;
  var qtyEl = drawerItem.querySelector('[data-qty-val]');
  if (qtyEl) qtyEl.textContent = qty;
}

// Sync a qty change FROM the drawer TO the cart page
function syncToCart(itemId, qty) {
  var cartItem = document.querySelector(
    '[data-cart-item][data-item-id="' + itemId + '"]'
  );
  if (!cartItem) return;
  var qtyEl = cartItem.querySelector('[data-qty-val]');
  if (qtyEl) qtyEl.textContent = qty;
}

// Item removal: animate both sides out, then recalculate
function removeItem(itemId) {
  var cartItem   = document.querySelector('[data-cart-item][data-item-id="'  + itemId + '"]');
  var drawerItem = document.querySelector('[data-drawer-item][data-item-id="' + itemId + '"]');

  if (cartItem)   cartItem.classList.add('is-removing');
  if (drawerItem) drawerItem.classList.add('is-removing');

  setTimeout(function() {
    if (cartItem)   cartItem.remove();
  }, 320);
  setTimeout(function() {
    if (drawerItem) drawerItem.remove();
    recalculate();
  }, 380);
}

Qty Stepper Behavior

// Qty stepper: minus button behavior
minusBtn.addEventListener('click', function() {
  var currentQty = parseInt(qtyDisplay.textContent);

  if (currentQty <= 1) {
    // At qty 1: trigger item removal (show trash icon)
    removeItem(itemId);
  } else {
    qtyDisplay.textContent = currentQty - 1;
    // Show minus symbol (unicode en-dash)
    minusBtn.textContent = '\u2212';
    syncToDrawer(itemId, currentQty - 1);
    recalculate();
  }
});

// Qty stepper: plus button behavior
plusBtn.addEventListener('click', function() {
  var currentQty = parseInt(qtyDisplay.textContent);
  if (currentQty >= 99) return; // max qty guard
  qtyDisplay.textContent = currentQty + 1;
  syncToDrawer(itemId, currentQty + 1);
  recalculate();
});

Drawer Open / Close

function openDrawer() {
  drawer.classList.add('is-open');
  backdrop.classList.add('is-open');
  document.body.style.overflow = 'hidden';
  drawer.setAttribute('aria-hidden', 'false');
}

function closeDrawer() {
  drawer.classList.remove('is-open');
  backdrop.classList.remove('is-open');
  document.body.style.overflow = '';
  drawer.setAttribute('aria-hidden', 'true');
}

// Close triggers
closeBtn.addEventListener('click', closeDrawer);
backdrop.addEventListener('click', closeDrawer);
document.addEventListener('keydown', function(e) {
  if (e.key === 'Escape') closeDrawer();
});

// Auto-open if URL has ?drawer=1 (used when navigating from PDP)
if (new URLSearchParams(window.location.search).get('drawer') === '1') {
  openDrawer();
}

Delivery Estimate Calculation

// Business day calculator: skips Sat (6) and Sun (0)
function addBizDays(date, days) {
  var d = new Date(date);
  var added = 0;
  while (added < days) {
    d.setDate(d.getDate() + 1);
    if (d.getDay() !== 0 && d.getDay() !== 6) added++;
  }
  return d;
}

var now     = new Date();
var earliest = addBizDays(now, 4); // 1 ship + 3 transit
var latest   = addBizDays(now, 7); // 2 ship + 5 transit

var months = ['Jan','Feb','Mar','Apr','May','Jun',
              'Jul','Aug','Sep','Oct','Nov','Dec'];

var deliveryStr =
  months[earliest.getMonth()] + ' ' + earliest.getDate() +
  ' – ' +
  months[latest.getMonth()]   + ' ' + latest.getDate();

document.getElementById('delivery-estimate').textContent =
  'Estimated delivery: ' + deliveryStr;

In a Shopify Liquid context, the delivery estimate can be rendered server-side using Liquid date filters — but the JS approach handles timezone and business-day logic more reliably without requiring custom app logic.

// IntersectionObserver: show sticky bar when CTA scrolls out of viewport
var ctaBtn = document.getElementById('checkout-cta');
var stickyBar = document.getElementById('sticky-checkout-bar');

if (ctaBtn && stickyBar) {
  var observer = new IntersectionObserver(function(entries) {
    entries.forEach(function(entry) {
      stickyBar.classList.toggle('is-visible', !entry.isIntersecting);
      stickyBar.setAttribute('aria-hidden', entry.isIntersecting ? 'true' : 'false');
    });
  }, { threshold: 0 });

  observer.observe(ctaBtn);
}

Accessibility Notes

  • aria-hidden="true" on drawer when closed; "false" when open
  • aria-label on all icon-only buttons: trash, modal close, gift info (i) on /cart page
  • role="dialog" on the drawer element; aria-label="Shopping bag"
  • Escape key closes drawer and all modals — handled by single keydown listener
  • Semantic HTML throughout: <button> for actions, <a> for navigation, <label> wrapping checkboxes
  • Screen reader text for status dots: wrap the dot in <span aria-hidden="true"> and let the label text speak for itself
  • Qty steppers are keyboard-operable native <button> elements — no custom ARIA needed
  • Promo panel toggle uses aria-expanded and aria-controls; panel uses hidden attribute (not display:none via CSS class) for correct AT behavior
